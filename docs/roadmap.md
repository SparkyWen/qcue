# Roadmap (Public)

A high-level, public-safe view of where QCue is heading. This roadmap is intentionally broad: it does
**not** include private business plans, confidential partnerships, internal launch timelines, or
commercial details. Items are directional, not commitments, and may change.

## Now

- Establish QCue as a clean, well-governed open-source project (AGPL-3.0, security policy,
  contribution workflow).
- Make the product easy to run locally end to end via stub mode (keyless, offline).
- Improve developer onboarding documentation for external contributors.

## Next

- A generic, self-host-oriented setup guide for the backend (local Postgres/Redis bootstrap and
  environment configuration), without exposing any private infrastructure.
- Broaden and document provider coverage in the harness.
- Strengthen the public test and CI story so contributions are easy to validate.
- Complete and publish the third-party dependency-license audit
  (see [`THIRD_PARTY_NOTICES.md`](THIRD_PARTY_NOTICES.md)).

## Later

- Deepen the idea engine (recall quality, consolidation).
- Expand the app experience across platforms.
- Community-driven extensions and integrations.

## How to influence the roadmap

Open an issue to propose or discuss a direction, or comment on existing ones. For anything large,
discuss before implementing (see [`../CONTRIBUTING.md`](../CONTRIBUTING.md)).
